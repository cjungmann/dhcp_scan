NAME

dhcp_scan − Scan journalctl output for a dhcp leases report.

SYNOPSIS

dhcp_scan [-h] [-s] [-i FILE] [-o DISPLAY_MODE]

DESCRIPTION

This utility can be used to prepare a more complete report of IP address leases than can be found in the dhcpd.leases file. Simply pipe the output of journalctl to dhcp_scan to prepare the report. See the EXAMPLES section below.

Note that it may take many seconds to scan all the journalctl records before the utility will be able to generate a report. For this reason, the default and recommended DISPLAY MODE is interactive so the user can switch between multiple views without having to rescan the records.

The default setting is to show the DHCP message names as they are encountered to reassure the user that the program has not frozen.

OPTIONS

-h

Show a brief help display.

-s

Silently scan journalctl records.

-i FILE

DHCP messages log file to scan. If this option is omitted, the program will read records from STDIN.

-o OUTPUT_MODE

The options are:
interactive

The default display mode. After scanning the records, the user selects a display mode for viewing the data. The report is shown in less. Leaving less returns the user to the mode selection menu.

ip

Prints report in IP address order and exits.

mac

Prints report in MAC address order and exits.

raw

Prints complete address negotiations in chronological order of the time of resolution, then exits.

EXAMPLES

With so many options for the journalctl utility, it makes more sense to have the user determine how to invoke it to generate the raw records needed for using dhcp_scan. Either use the journalctl output directly by piping it into dhcp_scan, or create a file that can be read with the -i option.

sudo journalctl -S today -u isc-dhcp-server | dhcp_scan

or

sudo journalctl -S today -u isc-dhcp-server > dhcpd.log
dhcp_scan -i dhcpd.log
